# 沉浸式写作 IDE - 团队级分布式部署与安全架构方案
(Immersive Writing IDE - Global Deployment & Security Architecture Plan)

> **文档说明**：本备忘录用于规划未来的“团队级私有化网络部署”。核心诉求是在确保**绝对的外部数据安全**与**极简的内部使用体验（无密码、无感登录）**之间取得完美平衡。本方案目前处于草案（Draft）阶段，待评估成熟后将按此指导落地。

---

## 一、 核心设计理念 (Core Philosophy)

1. **算存分离 (Compute & Storage Separation)**：将昂贵的显卡算力、便宜的逻辑处理、核心的商业数据完全拆分。实现随时关停省钱、一键打包迁移。
2. **内部零摩擦 (Internal Frictionless UX)**：去中心化，移除系统内部繁琐的多账户和登录验证墙。团队成员在一个统一的界面下，利用 `Project / 项目`（原 Folder）进行纯粹的分类归档协作。
3. **外部铜墙铁壁 (External Bulletproof Security)**：将防御阵线前置到“大门外”，利用顶级安保系统拦截一切非授权的互联网流量，隐匿服务器物理真实 IP。

---

## 二、 “三体”微服务架构蓝图 (Architecture Blueprint)

### 1. 算力引擎舱：GPU 节点 (The Engine)
- **部署位置**：RunPod (带高端 GPU 的按需实例)
- **核心任务**：专属运行 `Gemma-4-31B` 语言模型。
- **运维策略**：高度弹性。由于极其烧钱，仅在团队需要进行重度生成时开启，不用时随时销毁或暂停，不影响任何历史数据。

### 2. 大脑中枢舱：CPU 节点 (The Brain)
- **部署位置**：RunPod (极低成本的纯 CPU 实例)
- **核心任务**：运行经过 UX 爆改的 `Immersive Writing Engine` (原 Open-WebUI) 前后端程序。
- **持久化存储 (The Vault)**：将 RunPod 的 **Network Volume (网络硬盘)** 挂载至此节点。所有的 `open-webui.db`（历史对话）、记忆抽屉、全局碎片库统统存入此网盘。
  - *优势*：机器坏了/重置了都没关系，网盘不灭，数据永生。随时可以把网盘挂载到新的机器上秒级复活。

### 3. 隐形大门：Cloudflare 隧道与零信任网络 (The Gateway)
- **部署位置**：Cloudflare (全球云服务) + 自定义专属域名（如 `ai.yourteam.com`）
- **核心任务**：
  - **隧道直连 (Cloudflare Tunnels)**：在中枢舱 (CPU 节点) 挖一条加密地道直连 Cloudflare。不管 RunPod 重启后 IP 怎么变，域名永远稳定解析，且服务器公网 IP 绝对隐身。
  - **大门拦截 (Zero Trust Access)**：在域名上加装基于邮箱的白名单验证墙，阻挡一切黑客扫描，只允许指定员工进入。

## 三、 团队使用体验链路 (Team UX Flow)

### 阶段 1：首次登入 (验证期)
1. 团队成员在任何设备（电脑/手机）的浏览器输入团队专属域名。
2. 网页不会出现服务器，而是弹出 Cloudflare 极简白底的验证界面，要求输入邮箱。
3. 成员输入其工作邮箱（已被管理员加入白名单），立刻收到一封包含 PIN 码的邮件。
4. 输入 PIN 码，大门敞开。

### 阶段 2：日常使用 (免感期)
1. 验证通过后，在接下来设定的周期内（如 30 天或 90 天），该员工在自己的浏览器上输入域名，将**瞬间直达**沉浸式写作界面，中间没有任何阻力。
2. 界面内无账户概念（我们在代码层面已通过 `WEBUI_AUTH=False` 去除）。
3. 团队成员依靠左侧的 **“Project / 项目”** 树状图，来建立不同的小说卷章、素材库、设定集，实现直观的文档级协作。

### 阶段 3：外部黑客视角 (防御期)
- 任何人、机器人或搜索引擎如果知道这个域名并尝试访问，会被立刻截停在 Cloudflare 验证页。没有白名单内的邮箱，无法进行任何爆破，连网站内部 UI 都无法窥探。

## 四、 落地执行分步计划 (Implementation Steps)

- [ ] **Phase 1: 域名资产准备**
  - 购买团队专属的简短域名。
  - 将域名的 DNS 解析权托管给 Cloudflare（免费）。
- [ ] **Phase 2: 中枢舱与存储搭建**
  - 在 RunPod 建立网络硬盘 (Network Volume)。
  - 创建一个低配 CPU 实例，挂载该网盘，并在内部部署魔改版的项目代码。
- [ ] **Phase 3: 隐形隧道打通**
  - 在 CPU 实例的终端内，一键运行 Cloudflare Tunnel 的安装代码。
  - 在 Cloudflare 控制台将隧道绑定至自定义域名。
- [ ] **Phase 4: 零信任安保部署**
  - 在 Cloudflare Zero Trust 面板中创建 Access Application。
  - 配置 Policy（策略）：设定具体的邮箱白名单或邮箱后缀，并设置会话过期时间。
- [ ] **Phase 5: 算力引擎对接**
  - 启动 GPU 节点，将模型的 API 地址填入中枢舱的后台设置中，完成终极点火。

---
*注：本方案作为架构指导，随时可根据团队规模的变化和最新的技术栈（如 Cloudflare 政策更新）进行动态调整。*
